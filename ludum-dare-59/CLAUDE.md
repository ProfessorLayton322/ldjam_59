## Project Overview

Ludum Dare 59 entry ‚Äî a tower-defense‚Äìstyle game built in Godot 4.5 (GL Compatibility renderer). The project root for the Godot project is `ludum-dare-59/`; all `res://` paths are relative to that directory.

## Commands

Run the game headlessly (verify script registration, check for parse errors):

    cd ludum-dare-59
    godot --headless --editor --quit

Open the project in the editor (interactive):

    cd ludum-dare-59
    godot --editor

Regenerate start-direction SVG textures:

    cd ludum-dare-59/assets/textures
    python generate_start.py

## Architecture

### Scene Flow

```
menu.tscn  ‚Üí  demo.tscn   (Start button or Escape to quit)
           ‚Üí  settings.tscn  (Settings button; Escape or Back returns to menu)
```

`menu.tscn` is the main scene. All scene transitions use `get_tree().change_scene_to_file()`.

### Autoloads

- **ResolutionManager** (`scripts/resolution_manager.gd`) ‚Äî global singleton; persists resolution (Full HD / 4K) to `user://settings.cfg`. Other scripts call `ResolutionManager.set_resolution()` and listen to `resolution_changed`.
- **MCPGameBridge** (`addons/godot_mcp/game_bridge/mcp_game_bridge.gd`) ‚Äî addon only; ignore unless working on MCP tooling.

### Demo Scene (the game)

`demo.tscn` is a bare Node2D; everything is built procedurally in `scripts/demo.gd` (class `DemoScene`):

1. A `Graph` is built from a 5√ó3 grid of 64√ó64 px tiles at origin (100, 100).
2. Tile objects (`BaseTile` subclasses) are instantiated per vertex; SVG sprites are layered on top.
3. Node IDs 8, 9, 13, 14 become `CoreTile` (2√ó2 CPU block); IDs 0 and 10 become `SpawnerTile`.
4. `SpawnerTile` creates a timer that fires every `spawn_interval` seconds, instantiating enemies.
5. `Enemy` runs BFS over the `Graph` to find the shortest path to any `CpuVertex`, then tweens tile-by-tile; on arrival it calls `OnEnter` on the destination `CoreTile`, which calls `queue_free()` on the enemy.

### Graph / Data Model

`Graph` (Resource) and `GraphVertex` (RefCounted) live in `scripts/resources/` and are registered as global classes ‚Äî do not export their fields. Key invariants:

- Vertex ID: `id = y * width + x` (row-major).
- Position: tile centre = `origin + Vector2((x + 0.5) * tile_size.x, (y + 0.5) * tile_size.y)`.
- Neighbours stored in bottom, top, left, right order; missing neighbours are omitted.
- `Graph.build_from_grid()` is the primary constructor; `build_from_tilemap_layer()` / `build_from_level()` parse live TileMaps.

`CpuVertex` (Resource, exported `node_id`) marks goal nodes for enemy pathfinding.

### Tile Class Hierarchy

```
BaseTile (Node2D)       signals: triggered(source), entered(source)
‚îú‚îÄ‚îÄ CoreTile            OnEnter ‚Üí queue_free(source)
‚îú‚îÄ‚îÄ SpawnerTile         Timer ‚Üí OnTrigger ‚Üí spawn Enemy
‚îî‚îÄ‚îÄ TowerTile           (stub, not yet implemented)
```

Tiles are created as plain `new()` objects by `DemoScene._create_tile()`; visual sprites are added as children immediately after.

### Assets

SVG textures under `assets/textures/`:
- `tracks/` ‚Äî `pcb_empty`, `track_straight_v/h`, `track_corner_ne`, `track_t_junction`, `track_cross`
- `start/` ‚Äî directional spawner overlays; generated by `generate_start.py`
- `base/cpu.svg`, `enemies/circle_enemy.svg`

`scenes/tiles/track_*.tscn` are prefabs used by `scenes/example.tscn` (debug/test scene showing all track variants via TileMap).

## Graph Resource Notes

- `graph.gd` ‚Äî `Graph` Resource; `nodes: Array[GraphVertex]` is a plain var, intentionally not exported.
- `graph_vertex.gd` ‚Äî `GraphVertex` RefCounted; `id`, `position`, `neighbour_ids` are plain vars, not exported.
- `GraphVertex` name was chosen to avoid collisions with Godot's internal `Node`-adjacent names.

Keep these Godot-generated UID files:
- `ludum-dare-59/scripts/resources/graph.gd.uid`
- `ludum-dare-59/scripts/resources/graph_vertex.gd.uid`

## Verification

Verify global class registration:

    cd ludum-dare-59
    godot --headless --editor --quit

Expected: Godot registers global classes `Graph` and `GraphVertex`, exits 0. Unrelated `godot_mcp` warnings or shutdown errors are normal and not caused by graph scripts.
